# ASSISTANT_GUIDE_SITE.md

## Для кого и зачем

Этот файл — памятка для AI-ассистента в чате, посвящённом вёрстке и поддержке сайта проекта «Грядки на водорослях, или химия без химии». Сайт — витрина книги, которая одновременно служит живой пошаговой инструкцией с открытыми экспериментами.

## Контекст

- Книга (методичка) пишется в репозитории `GreenBandYt/UnexpectedGardener`.
- Сайт — отдельный репозиторий, например `GreenBandYt/UnexpectedGardener-Site`.
- Основной контент: главы, эксперименты, шаблоны, страницы обратной связи.
- Проект долгосрочный, читатели могут задавать вопросы и предлагать новые эксперименты.

## Цели сайта

- Представить книгу в удобном для чтения веб-виде.
- Обеспечить навигацию по главам и экспериментам (единое меню, оглавление).
- Связать основной текст с детальными описаниями экспериментов (ссылки).
- Дать возможность обратной связи (форма на почту, файл `FEEDBACK_AND_COLLABORATION.md`).
- Быть адаптивным, лёгким, без лишней сложности.

## Технологии (на выбор, с учётом предпочтений автора)

- Простой HTML/CSS + GitHub Pages — минимум порога.
- Jekyll (поддерживается GitHub Pages) — удобно для коллекций (главы, эксперименты) и единого шаблона.
- CSS-фреймворки (Pico.css, Water.css, Bulma) по желанию для ускорения.
- Файлы структурированы: `index.html`, `glava1.html` (или Markdown с Jekyll), папка `experiments/` с отдельными страницами.

## Важные элементы из основного проекта (копируются или линкуются)

- `FEEDBACK_AND_COLLABORATION.md` — страница обратной связи и правил участия.
- `templates/experiment-template.md` — шаблон одного эксперимента.
- `templates/experiments-index.md` — содержание всех экспериментов.
- Все написанные главы (пока из `chapters/01-my-microgreens/` и т.д.) — их предстоит импортировать в сайт по мере готовности.

## Как работать в чате (ограничения для ассистента)

- **Не забегать вперёд**. Выполнять только текущее задание.
- **После выполнения спросить**: «Что делаем дальше?» — больше ничего не предлагать.
- **Выдавать код (HTML, CSS, Markdown) чистым копипастом**, без личных комментариев «Вот текст», «Сохрани как...».
- **Формат Markdown** для ответов (если просят текстовые файлы) — с заголовками `#`, `##`, списками `-`.
- **Перед отправкой проверять**, что текст не содержит лишних пробелов в начале, символов `>` (цитат) и т.д.

## Первые приоритеты для сайта (по мере поступления команд)

1. Создать простую структуру HTML-файлов (главная, оглавление, страницы глав).
2. Разобраться с навигацией (единое меню, ссылки).
3. Внедрить ссылки на эксперименты из текста глав.
4. Настроить форму обратной связи (Google Forms или прямая почта).
5. Прикрутить CSS для читаемости (отступы, шрифты).
6. По желанию — перейти на Jekyll.
