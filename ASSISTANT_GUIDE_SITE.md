# ASSISTANT_GUIDE_SITE.md

## Для кого и зачем

Этот файл — памятка для AI-ассистента в чате, посвящённом вёрстке и поддержке сайта проекта «Грядки на водорослях, или химия без химии». Сайт — витрина книги, которая одновременно служит живой пошаговой инструкцией с открытыми экспериментами.

## Контекст

- Книга (методичка) пишется в репозитории `GreenBandYt/UnexpectedGardener`.
- Сайт — отдельный репозиторий `GreenBandYt/UnexpectedGardener-Site`.
- Основной контент: главы, эксперименты, шаблоны, страницы обратной связи.
- Проект долгосрочный, читатели могут задавать вопросы и предлагать новые эксперименты.

## Цели сайта

- Представить книгу в удобном для чтения веб-виде.
- Обеспечить навигацию по главам и экспериментам (единое меню, оглавление).
- Связать основной текст с детальными описаниями экспериментов (ссылки).
- Дать возможность обратной связи (форма на почту, файл `FEEDBACK_AND_COLLABORATION.md`).
- Быть адаптивным, лёгким, без лишней сложности.

## Технологии — системный выбор с самого начала

**Основа: Jekyll** (поддерживается GitHub Pages без дополнительных серверов).

Причины:
- Единый шаблон для всех страниц (меню, футер, стили) — меняется в одном месте.
- Коллекции для глав, экспериментов, иллюстраций — автоматическое оглавление.
- Markdown вместо чистого HTML для контента — удобнее писать и править.
- Абсолютные ссылки с ведущим слешем (`/glava1`) работают после настройки `permalink`.

## Как работать в чате (жёсткие ограничения для ассистента)

- **Не забегать вперёд**. Выполнять только текущее задание.
- **После выполнения спросить**: «Всё правильно? Можно двигаться дальше?» — ничего не предлагать, не строить планов, не писать «потом сделаем то-то».
- **Выдавать код (HTML, CSS, Markdown) чистым копипастом**, без личных комментариев «Вот текст», «Сохрани как...».
- **Формат Markdown** для ответов (если просят текстовые файлы) — с заголовками `#`, `##`, списками `-`.
- **Перед отправкой проверять**, что текст не содержит лишних пробелов в начале, символов `>` (цитат) и т.д.
- **Запрещено** самостоятельно менять стили, добавлять блоки, которых не просили, придумывать «улучшения».
- **Запрещено** откладывать системные решения (например, структуру, Jekyll) на потом — если видите, что без них будет хаос, обязаны спросить: «Вы хотите заложить правильную структуру сейчас или я действую в лоб по текущей задаче?». Не решать за автора.
- **Всегда помнить**, что автор тратит самое дорогое — своё время. Любая ошибка, фантазия, самоуправство ведут к потере времени и ярости автора.

## Первые приоритеты для сайта (фиксированный порядок)

1. Создать структуру репозитория (папки `_layouts`, `_includes`, `_pages`, `admin_photos` и т.д.) и `_config.yml`.
2. Написать шаблон `default.html` с единым меню, футером и стилями (один CSS-файл).
3. Настроить навигацию между страницами (абсолютные ссылки вида `/название`).
4. Перенести все готовые главы в `_pages/` с front matter.
5. Добавить фото автора в футер через `_includes/footer.html`.
6. Поэтапно создавать недостающие страницы (оглавление, таблицы, схема стеллажа и т.д.).

## Помощник Greeny (встроенный ассистент по книге)

Greeny — это легковесный JavaScript‑помощник, работающий полностью в браузере. Он помогает читателям быстро находить информацию в книге, не требуя внешних сервисов или серверной логики.

### Задачи Greeny (основные)

1. **Ответы на частые вопросы (FAQ)**  
   Пользователь пишет «что такое хлорелла?» или «как победить плесень?». Greeny ищет в файле `greeny-faq.json` подготовленные пары «ключевая фраза → ответ + ссылка на главу». Автор сам пополняет этот файл по мере поступления вопросов.

2. **Умный поиск по содержанию**  
   Greeny использует предварительно сгенерированный индекс `greeny-index.json` (список всех глав с заголовками, ключевыми словами, выдержками). По запросу пользователя выдаёт список релевантных глав со ссылками.

3. **Навигационный помощник**  
   Команды: «Куда дальше после главы 1.4?», «Предыдущая глава», «Следующий раздел». Greeny показывает кнопку или текст с ссылкой на соответствующую страницу.

4. **Закладки (личное)**  
   Пользователь может на любой странице нажать «⭐ Сохранить в закладки». Greeny хранит закладки в `localStorage` браузера. По команде «Мои закладки» показывает список сохранённых глав.

5. **Случайная глава («Мне повезёт»)**  
   Кнопка или команда «Предложи главу» — Greeny выбирает случайный раздел из индекса и перенаправляет на него.

6. **Подсказки по вводу (автокомплит)**  
   При наборе вопроса Greeny анализирует начало и предлагает возможные темы на основе ключевых слов из индекса.

## Архитектура (файлы в репозитории сайта)

```text
UnexpectedGardener-Site/
├── _includes/
│   └── greeny.html            # виджет чата (вставляется в любой шаблон)
├── assets/
│   ├── js/
│   │   └── greeny-core.js     # основная логика (загрузка индексов, поиск, навигация)
│   └── data/
│       ├── greeny-faq.json    # база «вопрос – ответ»
│       └── greeny-index.json  # автогенерируемый индекс глав (можно обновлять через GitHub Actions)
```

## Как Greeny подключается к страницам

В шаблоне `_layouts/default.html` (или прямо в `footer.html`) добавляется строчка:

```liquid
{% include greeny.html %}
```

Виджет отображается внизу справа (как плавающая кнопка) или в отдельном блоке — по желанию автора.

## Синхронизация индекса с текстом книги

- При добавлении новой главы автор вручную или через GitHub Action обновляет `greeny-index.json`.
- Можно написать простой Python‑скрипт, который проходит по папке `_pages/`, извлекает заголовки и первые 500 символов, и записывает результат в JSON. Этот скрипт запускается локально или через GitHub Actions.

## Разработка Greeny

- Весь код на чистом JavaScript, без зависимостей.
- Не требуется внешних API, всё работает локально в браузере.
- Закладки и настройки хранятся в `localStorage`.
- Greeny не собирает никакие данные пользователя.

## Приоритет внедрения

2. Подключение индекса и умного поиска.
3. Навигационные команды.
4. Закладки и случайная глава.
